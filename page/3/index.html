文件名和文件系统

  • inode 和 文件名 映射关系保存在 文件名目录 中，文件名 的变动不会修改文件inode信息
  • 通过 inode table 找到文件 元数据 和 文件内容， 包含 引用计数 等信息
  • i_count 是文件被进程引用计数，i_nlink 是硬链接(磁盘)引用计数， rm 命令减少 i_nlink， 当 i_count 和 i_nlink 均为0时，文件才被删除(inode 和 文件名 的映射被删除，实际磁盘上block数据块未删除)。

请注意，如果使用 rm 来删除文件，通常仍可以将该文件恢复原状。如果想保证该文件的内容无法还原，请考虑使用shred

调度

有三种方式指定 Pod 只运行在指定的 Node 节点上

  • nodeSelector：只调度到匹配指定 label 的 Node 上
  • nodeAffinity：功能更丰富的 Node 选择器，比如支持集合操作
  • podAffinity：调度到满足条件的 Pod 所在的 Node 上
  • Taints 和 tolerations：Taints 和 tolerations 用于保证 Pod 不被调度到不合适的 Node 上，其中 Taint 应用于 Node 上，而 toleration 则应用于 Pod 上
  • 优先级调度

调度工作原理

kube-scheduler 调度分为两个阶段，predicate 和 priority

  • predicate：过滤不符合条件的节点
  • priority：优先级排序，选择优先级最高的节点

代码入口：pkg/scheduler，cmd/kube-scheduler

go hashmap

发表于 2020-06-10 更新于 2022-11-06 分类于 go
  • hash0保存1个随机hash种子
  • 每个桶保存8个k/v对
  • 每个桶内存最后8位为指向下一个桶的指针
  • map创建时，若桶数量>2^4，在extra中创建备用的溢出桶，与正常桶内存连续
  • 插入时，若所有正常桶和溢出桶已满，使用extra中备用的next溢出桶插入key，并加到正常桶末尾的overflow指针中，调用typedmemmove内存拷贝key到桶内地址
  • 扩容因子超过6.5，翻倍扩容
  • 扩容因子未超过6.5，已有太多溢出桶，通常是不断插入元素后，一次delete删除所有元素，此时存在缓慢内存泄漏，采用等量扩容
  • 扩容期间，使用oldbuckets提供查询，插入和删除操作会触发旧桶到新桶的分流，所有分流完成后清空旧桶
type hmap struct {
    count     int // 元素数量
    flags     uint8  // 用于基本并发检测，抛出并发异常
    B         uint8  // len(buckets) == 2^B
    noverflow uint16
    hash0     uint32 // hash seed，fastrand()

    buckets    unsafe.Pointer // 当前buckets指针
    oldbuckets unsafe.Pointer // 扩容时保存旧buckets指针
    nevacuate  uintptr        // progress counter for evacuation (buckets less than this have been evacuated)

    extra *mapextra // optional fields
}

type mapextra struct {
    overflow    *[]*bmap
    oldoverflow *[]*bmap

    // nextOverflow holds a pointer to a free overflow bucket.
    nextOverflow *bmap
}

// 1个桶保存8个k/v键值对
type bmap struct {
    tophash [bucketCnt]uint8
}

const (
    // Maximum number of key/elem pairs a bucket can hold.
    bucketCntBits = 3
    bucketCnt     = 1 << bucketCntBits
)

冲突解决

  • 开放寻址法： 发生冲突时写入下一个不为空的位置，hash查询不匹配时顺序查找后面位置元素
  • 链表/红黑树法： hash定位桶，每个桶为链表/红黑树

初始化

cmd/compile/internal/gc.maplit

  • 字面变量
  • 运行时make

make流程：

  1. 计算哈希占用的内存是否溢出或者超出能分配的最大值
  2. 调用 fastrand 获取一个随机的哈希种子
  3. 根据传入 hint 计算最小需要的桶的数量
  4. 创建用于保存桶的数组
    4.1 桶数量<2^4，不使用溢出桶，避免过度计算
    4.2 桶数量>=2^4，额外创建2^4-B个溢出桶，和正常buckets在内存中连续

查询

v     := hash[key] // => v     := *mapaccess1(maptype, hash, &key)
v, ok := hash[key] // => v, ok := mapaccess2(maptype, hash, &key)

NOTE: The returned pointer may keep the whole map live, so don’t hold onto it for very long.

mapaccess1流程：

  1. 高8位计算hash，与bmap的8个桶比较
  2. 循环遍历正常桶和溢出桶(内存连续)
// overflow指针指向下一个bmap
func (b *bmap) overflow(t *maptype) *bmap {
    return *(**bmap)(add(unsafe.Pointer(b), uintptr(t.bucketsize)-sys.PtrSize))
}

mapaccess2: 略

插入

m[k] = v，在compile阶段编译为runtime.mapassign获得v指针进行赋值

mapassign流程：

  1. 根据key计算hash和桶
  2. 遍历正常桶和溢出桶，依次判断tophash和完整key
    2.1 如果找到相同key，返回对应val地址
    2.2 如果遍历到空key，说明没有相同key，在空key位置new新k/v对，返回val地址
    2.3 如果所有桶已满，仍没找到相同key，从extra中取一个备用溢出桶加入正常桶末尾overflow指针。若所有备用溢出桶也用满，分配一个新bmap使用。

删除

逻辑同插入，将key设为emptyOne，略

扩容

扩容判断条件：

  1. 装在因子超过6.5，翻倍扩容，通过mask将旧桶数据分流到2个新桶上
  2. 有太多溢出桶，等量扩容，evacDst结构体只会初始化1个，旧桶和新桶一对一

扩容期间oldbuckets保存旧桶，查询使用oldbuckets，插入和删除会触发分流，在所有旧桶分流到新桶后，清空oldbuckets和oldoverflow

  • dataqsiz=0表示同步chan，dataqsiz>0表示异步chan
  • recvq / sendq其中之一必定为空，或两者都为空
  • qcount > 0表示recvq为空
  • qcount < dataqsiz表示sendq为空
type hchan struct {
    qcount   uint           // buffer中元素数量
    dataqsiz uint           // make chan时候指定的buf大小
    buf      unsafe.Pointer // buffer
    elemsize uint16         // 每个元素大小
    elemtype *_type         // 元素类型
    closed   uint32         // chan是否已关闭，=0表示未关闭
    sendx    uint           // send索引下标
    recvx    uint           // recv索引下标
    recvq    waitq          // 接受等待队列，双向链表
    sendq    waitq          // 发送等待队列，双向链表

    lock mutex
}

chansend流程：

  1. 若recvq不为空，表示有等待的recv sudog，调用memmove进行数据拷贝，然后调用goready唤醒该recv sudug
  2. 若recvq为空
    2.1 若qcount < dataqsiz，表示当前chan还有缓存空间，调用memmove进行数据拷贝
    2.2 若缓存空间满，将当前gp封装成sudog放入sendq队列，并进入休眠(在recv函数中进行数据拷贝，然后调用goready被唤醒)，唤醒后数据已被拷贝走，release释放sudog

chanrecv流程：

  1. 若sendq不为空，表示有等待的send sudog。若是同步chan，将send sudo拷贝到recv sudo；若是带缓存的chan，将缓存head元素传给recv sudo，将sendq取出元素传给缓存队尾，然后调用goready唤醒send sudug
  2. 若sendq为空
    2.1 若创建时指定的buf大小>0，且缓存中有数据，从缓存中取第一个元素传给recv sudo
    2.2 若都不满足，进入recvq队列

close流程：

  1. 设置closed = 1
  2. 取出所有sendq和recvq中sudog，调用goready唤醒所有g

内存结构

go基于TCMalloc，将页划分为67个等级，页最小单位8K，定义在sizeclasses.go，大小从0～32K。通过定义常量数组，将内存分配时大小转换的时间复杂度降为O(1)
mspan是某个class页的集合，包含页起始地址、页数、页等级、已分配对象数等。mspan是一个双向链表。
每个P保存本地mcache，mcache是由2*67个mspan组成的数组，每个等级分为scan和noscan两种类型。为0～32K大小对象分配内存直接使用mcache池。
noscan类型的mspan上分配不包含指针的对象，在GC时这些对象无需遍历确认，可以直接释放回收。
mcentral是mspan的包装，每个mcentral包含1个mspan双向空链表和1个mspan双向非空链表，不同P请求同一个mcentral需要竞争锁。空链表包含没有空闲对象的mspan或缓存在mcache中的mspan，当mspan被释放时放进非空链表，非空链表包含至少有1个空闲对象的mspan。通过mcentral请求新mspan时，从非空链表取出mspan放入空链表。
arena结构是mspan的位图，目的是高效分配内存。1个arean大小为heapArenaBytes=64MB
mheap是全局内存对象，保存67个页等级的mcentral数组，每个级别的mcentral有自己的锁，因此P1申请2K的mspan和P2申请16K的mspan可以并行执行。

每个class的mspan含有的page数不同，具体见sizeclasses.go，比如：

  • class 43的mspan用来分配4K大小的对象，含有1个8K page，共可以分配2个对象，全分配后内存可以全部用完
  • class 44的mspan用来分配4864B大小的对象，含有3个8K page，共可以分配5个对象，全分配后尾部有256B浪费

0值对象返回zerobase指针
小对象分配顺序：per-P本地mcache -> mcentral -> mheap -> OS底层调用mmp
tiny对象优先从mspan.tiny分配

    c := getg().m.mcache
    noscan := typ == nil || typ.ptrdata == 0

    if size <= maxSmallSize {

            if off+size <= maxTinySize && c.tiny != 0 {

                return x

  2. 若本地相应class的mcache对象已满，调用mspan.refill从mheap相应class的central申请，需加解锁

    2.2 从空链表(没有空闲对象或在mcache中)取出状态为需sweep的mspan并执行sweep

  4. 若mheap内存满，调用mheap.grow，从os申请内存，底层调用mmp
func (c *mcache) nextFree(spc spanClass) (v gclinkptr, s *mspan, shouldhelpgc bool) {
    s = c.alloc[spc]
    freeIndex := s.nextFreeIndex()
    if freeIndex == s.nelems {
        // The span is full.
        c.refill(spc)
        s = c.alloc[spc]
    }
    s.allocCount++
    return
}

func (c *mcache) refill(spc spanClass) {
    s := c.alloc[spc]

    if s != &emptymspan {
        // Mark this span as no longer cached.
        atomic.Store(&s.sweepgen, mheap_.sweepgen)
    }
    // Get a new cached span from the central lists.
    s = mheap_.central[spc].mcentral.cacheSpan()
    if s == nil {
        throw("out of memory")
    }

    // Indicate that this span is cached and prevent asynchronous
    // sweeping in the next sweep phase.
    s.sweepgen = mheap_.sweepgen + 3

    c.alloc[spc] = s
}

func (c *mcentral) cacheSpan() *mspan {
    lock(&c.lock)
    sg := mheap_.sweepgen
retry:
    var s *mspan
    for s = c.nonempty.first; s != nil; s = s.next {
        if s.sweepgen == sg-2 && atomic.Cas(&s.sweepgen, sg-2, sg-1) {
            c.nonempty.remove(s)
            c.empty.insertBack(s)
            unlock(&c.lock)
            s.sweep(true)
            goto havespan
        }
        if s.sweepgen == sg-1 {
            // the span is being swept by background sweeper, skip
            continue
        }
        // we have a nonempty span that does not require sweeping, allocate from it
        c.nonempty.remove(s)
        c.empty.insertBack(s)
        unlock(&c.lock)
        goto havespan
    }

    for s = c.empty.first; s != nil; s = s.next {
        if s.sweepgen == sg-2 && atomic.Cas(&s.sweepgen, sg-2, sg-1) {
            // we have an empty span that requires sweeping,
            // sweep it and see if we can free some space in it
            c.empty.remove(s)
            // swept spans are at the end of the list
            c.empty.insertBack(s)
            unlock(&c.lock)
            s.sweep(true)
            freeIndex := s.nextFreeIndex()
            if freeIndex != s.nelems {
                s.freeindex = freeIndex
                goto havespan
            }
            lock(&c.lock)
            // the span is still empty after sweep
            // it is already in the empty list, so just retry
            goto retry
        }
        if s.sweepgen == sg-1 {
            // the span is being swept by background sweeper, skip
            continue
        }
        // already swept empty span,
        // all subsequent ones must also be either swept or in process of sweeping
        break
    }
    unlock(&c.lock)

    // Replenish central list if empty.
    s = c.grow()
    if s == nil {
        return nil
    }
    lock(&c.lock)
    c.empty.insertBack(s)
    unlock(&c.lock)

    // At this point s is a non-empty span, queued at the end of the empty list,
havespan:
    // ...

    return s
}

对象回收流程

  1. 大对象的mspan中对象全部释放时，mspan归还给mheap
  2. 小对象的mspan从空链表移到非空链表，供mcentral.cacheSpan中获取mspan；如果mspan中对象全部释放，直接归还到mheap
func (s *mspan) sweep(preserve bool) bool {
    // 有对象释放&&小对象
    if nfreed > 0 && spc.sizeclass() != 0 {
        res = mheap_.central[spc].mcentral.freeSpan(s, preserve, wasempty)
        // 大对象&&全部释放
    } else if freeToHeap {
        // Free large span to heap
        mheap_.freeSpan(s)
        res = true
    }
    if !res {
        // The span has been swept and is still in-use, so put it on the swept in-use list.
        mheap_.sweepSpans[sweepgen/2%2].push(s)
    }
    return res
}

func (c *mcentral) freeSpan(s *mspan, preserve bool, wasempty bool) bool {
    lock(&c.lock)

    if wasempty {
        c.empty.remove(s)
        c.nonempty.insert(s)
    }

    atomic.Store(&s.sweepgen, mheap_.sweepgen)

    if s.allocCount != 0 {
        unlock(&c.lock)
        return false
    }

    c.nonempty.remove(s)
    unlock(&c.lock)
    mheap_.freeSpan(s)
    return true
}

GC

gcpercent通过环境变量GOGC设置，用于控制GC速度，若无环境变量默认100%，若环境变量为”off”，会关闭2分钟1次的强制GC
函数gcSetTriggerRatio根据当次GC的速度调整下一次GC的速度，初始速度7/8，最小0.6，最大0.95。计算方式如下：

mark phase阈值： float64(memstats.heap_marked) * (1 + triggerRatio)，如果有p正在sweep需求更大内存，自动上调
next GC阈值： memstats.heap_marked + memstats.heap_marked*uint64(gcpercent)/100，内存扩大一倍启动GC，若小于mark phase阈值自动上调

GC通过以下逻辑触发：

  1. heap上涨触发了GC，每次分配对象需要新申请mspan，触发sweep后，最后会调用test判断是到达heap阈值否需要GC
  2. 在sysmon的每一次调度中，都会调用test判断距离上一次GC是否超过2分钟达到强制GC阈值
  3. gcTriggerCycle为runtime.GC()手动调用GC case
func (t gcTrigger) test() bool {
    if !memstats.enablegc || panicking != 0 || gcphase != _GCoff {
        return false
    }
    switch t.kind {
    case gcTriggerHeap:
        return memstats.heap_live >= memstats.gc_trigger
    case gcTriggerTime:
        if gcpercent < 0 {
            return false
        }
        lastgc := int64(atomic.Load64(&memstats.last_gc_nanotime))
        return lastgc != 0 && t.now-lastgc > forcegcperiod
    case gcTriggerCycle:
        return int32(t.n-work.cycles) > 0
    }
    return true
}

三色标记法：

第一阶段：

  1. 每个P启动一个goroutine，进入park，用于第二阶段调用gcDrain消费gcw
  2. 初始reset为白色，stop the world，完成未完成的sweep
  3. 启动gc write barriers，初始化root节点的扫描工作任务，在第二阶段中循环所有root节点执行markroot
  4. 对p.mcache和p.tiny标记灰色
  5. start the world
func gcStart(trigger gcTrigger) {
    gcBgMarkStartWorkers() // 每个P启动一个goroutine，进入park，用于第二阶段调用gcDrain消费gcw

    systemstack(gcResetMarkState) // 初始reset为白色

    systemstack(stopTheWorldWithSema) // stop the world
    systemstack(func() {
        finishsweep_m()
    })

    setGCPhase(_GCmark) // 启动gc write barriers

    gcBgMarkPrepare() // Must happen before assist enable.
    gcMarkRootPrepare() // 计算work.markrootJobs

    gcMarkTinyAllocs() // 对p.mcache和p.tiny标记灰色

    // Concurrent mark.
    systemstack(func() {
        now = startTheWorldWithSema(trace.enabled) // start the world
    })
}

第二阶段：

  1. markroot对所有root节点做标记，包含几个特殊任务随机分配给p执行，比如：释放全局sched.gFree.stack的栈，该队列保存有栈的dead G，释放栈后放入全局sched.gFree.noStack
    1.1 默认每个p扫描自己的栈上引用对象，包括栈上的defer链表。若为noscan对象直接标记黑色，否则根据对象引用进行标记。

第三阶段：

gcMarkDone先stop the world，将写屏障记录的所有修改的指针加入到标记队列，完成后start the world

第四阶段：

gcSweep释放内存

GC停顿时间主要在第一阶段和第三阶段，第二阶段为和用户代码并行执行。第一次停顿用于扫描root节点，第二次停顿用于追加写屏障中记录。GC停顿时间主要消耗为root节点数量，和heap总大小无直接关系。

举个栗子：

  1. 初始状态下所有对象都是白色的。
  2. 接着开始扫描根对象a、b; 由于根对象引用了对象A、B,那么A、B变为灰色对象，接下来就开始分析灰色对象，分析A时，A没有引用其他对象很快就转入黑色，B引用了D，则B转入黑色的同时还需要将D转为灰色，进行接下来的分析。
  3. 灰色对象只有D，由于D没有引用其他对象，所以D转入黑色。标记过程结束
  4. 最终，黑色的对象会被保留下来，白色对象会被回收掉。

每个p有写屏障buf，用于避免第二阶段中用户代码修改引用导致gc漏标记。

  • 若没有spin机制，锁一旦释放，当前运行g会更大概率得到锁。若锁的时间很短，没有spin机制的情况下得不到锁的g都需要休眠park，会增加调度开销，存在长尾
  • 在默认模式下，所有g都会先spin，waiter计数，然后插入链表尾并进入休眠，底层调用park函数
  • 解锁后从FIFO链表头唤醒1个g，若该g等待时间>1ms会尝试进入饥饿模式，此时仍可能有新g竞争锁。每次循环都会尝试CAS state到饥饿模式，若CAS失败会直接插入链表头，下次第一个被唤醒。
  • 进入饥饿模式后，所有新g竞争时不会spin也不会尝试加锁，直接插入链表尾。每次解锁后从链表头唤醒1个g消费锁，若该g等待时间不超过1ms，退出饥饿模式。
type Mutex struct {
	state int32
	sema  uint32
}

const (
	mutexLocked = 1 << iota // 0001
	mutexWoken              // 0010
	mutexStarving           // 0100
	mutexWaiterShift = iota // 0011

	starvationThresholdNs = 1e6 // 超过1ms进入饥饿模式
)

func (m *Mutex) Lock() {
	if atomic.CompareAndSwapInt32(&m.state, 0, mutexLocked) {
		return
	}

	m.lockSlow()
}

func (m *Mutex) lockSlow() {
	var waitStartTime int64
	starving := false
	awoke := false
	iter := 0
	old := m.state
	for {
		// 判断是否为饥饿模式
		if old&(mutexLocked|mutexStarving) == mutexLocked && runtime_canSpin(iter) {
			// 如果未设置awoke且waiter计数>0，说明其他m在等待锁，cas设置awoke
			if !awoke && old&mutexWoken == 0 && old>>mutexWaiterShift != 0 &&
				atomic.CompareAndSwapInt32(&m.state, old, old|mutexWoken) {
				awoke = true
			}
			runtime_doSpin() // 自旋30次
			iter++
			old = m.state
			continue
		}
		new := old
		// 非starving模式，设置new的locked
		if old&mutexStarving == 0 {
			new |= mutexLocked
		}
		// 已上锁或饥饿状态，waiter计数+1
		if old&(mutexLocked|mutexStarving) != 0 {
			new += 1 << mutexWaiterShift
		}
		// 处于饥饿模式并且已上锁，将starving置1
		if starving && old&mutexLocked != 0 {
			new |= mutexStarving
		}
		if awoke {
			new &^= mutexWoken // 如果当前g是被唤醒的，清除woken
		}
		// 如果state!=old，说明有其他g抢锁，重新进入for循环
		if atomic.CompareAndSwapInt32(&m.state, old, new) {
			// 如果old没上锁且是普通模式，抢锁成功，退出
			if old&(mutexLocked|mutexStarving) == 0 {
				break
			}
			queueLifo := waitStartTime != 0 // 第一次循环固定false
			if waitStartTime == 0 {
				waitStartTime = runtime_nanotime()
			}
			runtime_SemacquireMutex(&m.sema, queueLifo, 1)
			starving = starving || runtime_nanotime()-waitStartTime > starvationThresholdNs
			old = m.state
			if old&mutexStarving != 0 {
				delta := int32(mutexLocked - 1<<mutexWaiterShift)
				// 如果在饥饿模式并且自己是最后一个，清除starving标示
				if !starving || old>>mutexWaiterShift == 1 {
					delta -= mutexStarving
				}
				atomic.AddInt32(&m.state, delta)
				break
			}
			awoke = true
			iter = 0
		} else {
			old = m.state
		}
	}
}

type RWMutex struct {
	w           Mutex  // held if there are pending writers
	writerSem   uint32 // semaphore for writers to wait for completing readers
	readerSem   uint32 // semaphore for readers to wait for completing writers
	readerCount int32  // number of pending readers
	readerWait  int32  // number of departing readers
}

const rwmutexMaxReaders = 1 << 30
func (rw *RWMutex) RLock() {
	if atomic.AddInt32(&rw.readerCount, 1) < 0 {
		// readerSem叶链表尾=当前g
		runtime_SemacquireMutex(&rw.readerSem, false, 0)
	}
}

func (rw *RWMutex) RUnlock() {
	if r := atomic.AddInt32(&rw.readerCount, -1); r < 0 {
		// 有w在等锁
		rw.rUnlockSlow(r)
	}
}

func (rw *RWMutex) rUnlockSlow(r int32) {
	if atomic.AddInt32(&rw.readerWait, -1) == 0 {
		// 最后一个r锁释放，从writerSem叶链表头唤醒等待的w锁
		runtime_Semrelease(&rw.writerSem, false, 1)
	}
}

func (rw *RWMutex) Lock() {
	rw.w.Lock()
	r := atomic.AddInt32(&rw.readerCount, -rwmutexMaxReaders) + rwmutexMaxReaders // readerCount设一个大负数
	// Wait for active readers.
	if r != 0 && atomic.AddInt32(&rw.readerWait, r) != 0 {
		runtime_SemacquireMutex(&rw.writerSem, false, 0)
	}
}

func (rw *RWMutex) Unlock() {
	r := atomic.AddInt32(&rw.readerCount, rwmutexMaxReaders) // 还原readerCount设置的大负数
	// readerSem叶链表头->尾循环唤醒等待的r
	for i := 0; i < int(r); i++ {
		runtime_Semrelease(&rw.readerSem, false, 0)
	}
	rw.w.Unlock()
}

  • lock流程
  1. m.lock++
  2. 原子load key，如果没锁，尝试原子cas key->locked，cas成功则返回，否则自旋30次，多核下一共尝试4遍
  3. 其他m占着锁，此时key=locked，设置m.nextwaitm = muintptr(0)，for循环不断load key并尝试原子cas key->g.m，将自己设为等待锁的m
    3.1 如果cas失败，可能还有其他m已将key cas为自己，继续循环；也可能是锁已释放，跳到1.重新判断锁
    3.2 如果cas成功，m进入等待
  • unlock流程
  1. 如果key=locked，尝试原子cas key->0解锁，成功则返回
  2. 如果key!=locked，有其他m在等待锁，获取等待的m,尝试原子cas key->m.nextwaitm并唤醒等待的m
  3. 当前m.lock–，如果lock=0并且可抢占，设置抢占标记
type mutex struct {
	key uintptr
}
const (
	locked uintptr = 1

	active_spin     = 4 // 多核loop 4次
	active_spin_cnt = 30 // 每次loop自旋30次
	passive_spin    = 1
)

func lock(l *mutex) {
	gp := getg()
	gp.m.locks++

	if atomic.Casuintptr(&l.key, 0, locked) {
		return
	}
	semacreate(gp.m) // 创建OS信号量

	spin := 0
	if ncpu > 1 {
		spin = active_spin // 多核loop 4次
	}
Loop:
	for i := 0; ; i++ {
		v := atomic.Loaduintptr(&l.key)
		if v&locked == 0 {
			// Unlocked. Try to lock.
			if atomic.Casuintptr(&l.key, v, v|locked) {
				return
			}
			i = 0
		}
		if i < spin {
			procyield(active_spin_cnt) // 自旋30次
		} else if i < spin+passive_spin {
			osyield()
		} else {
            // 其他g已获得lock，gp.m.nextwaitm->muintptr(0)，l.key->gp.m
			for {
				gp.m.nextwaitm = muintptr(v &^ locked)
				if atomic.Casuintptr(&l.key, v, uintptr(unsafe.Pointer(gp.m))|locked) {
					break
				}
				v = atomic.Loaduintptr(&l.key)
				if v&locked == 0 {
					continue Loop
				}
			}
			if v&locked != 0 {
				// Queued. Wait.
				semasleep(-1)
				i = 0
			}
		}
	}
}

func unlock(l *mutex) {
	gp := getg()
	var mp *m
	for {
		v := atomic.Loaduintptr(&l.key)
		if v == locked {
			if atomic.Casuintptr(&l.key, locked, 0) {
				break
            }
        // l.key是其他等待m
		} else {
            // 获取等待m
            mp = muintptr(v &^ locked).ptr()
            // key换为等待m.nextwaitm
			if atomic.Casuintptr(&l.key, v, uintptr(mp.nextwaitm)) {
				// Dequeued an M.  Wake it.
				semawakeup(mp)
				break
			}
		}
	}
	gp.m.locks--
	if gp.m.locks == 0 && gp.preempt { // 重置抢占标示，在newstack栈分配中判断
		gp.stackguard0 = stackPreempt
	}
}

func (p *Pool) Put(x interface{}) {
    if x == nil {
        return
    }
    if race.Enabled {
        if fastrand()%4 == 0 {
            // Randomly drop x on floor.
            return
        }
        race.ReleaseMerge(poolRaceAddr(x))
        race.Disable()
    }
    l, _ := p.pin()
    if l.private == nil {
        l.private = x
        x = nil
    }
    if x != nil {
        l.shared.pushHead(x)
    }
    runtime_procUnpin()
    if race.Enabled {
        race.Enable()
    }
}

func (p *Pool) pin() (*poolLocal, int) {
    pid := runtime_procPin() // 在runtime_procUnpin前当前G和P不会被抢占，防止下面pid发生改变
    s := atomic.LoadUintptr(&p.localSize)
    l := p.local
    // pid < p.local数组长度时，在p.local里查找poolLocal对象
    if uintptr(pid) < s {
        return indexLocal(l, pid), pid
    }
    // 否则调用pinSlow创建新poolLocal
    return p.pinSlow()
}

    runtime_procUnpin() // 先解除抢占再lock
    allPoolsMu.Lock()
    defer allPoolsMu.Unlock()

    s := p.localSize
    l := p.local
    if uintptr(pid) < s {
        return indexLocal(l, pid), pid // 解除抢期间可能p.local已发生变化

    atomic.StoreUintptr(&p.localSize, uintptr(size))
    return &local[pid], pid

// 1。调用pin获取poolLocal和pid
// 2. poolLocal.private为空，依次尝试以下方式
// 2.1 从poolLocal.shared队列中获取对象
// 2.2 调用getSlow从其他P偷取对象
// 2.3 调用new创建对象
// 3. poolLocal.private不为空，直接复用对象返回
func (p *Pool) Get() interface{} {
    if race.Enabled {
        race.Disable()
    }
    l, pid := p.pin()
    x := l.private // poolLocal.private不为空时，复用对象返回
    l.private = nil
    if x == nil {
        // poolLocal.private为空，从poolLocal.shared队列中获取对象
        x, _ = l.shared.popHead()
        if x == nil {
            x = p.getSlow(pid)
        }
    }
    runtime_procUnpin()
    if race.Enabled {
        race.Enable()
        if x != nil {
            race.Acquire(poolRaceAddr(x))
        }
    }
    if x == nil && p.New != nil {
        x = p.New() // 用new()函数创建对象
    }
    return x
}

// 当本地P无可用对象，从其他P偷取对象
func (p *Pool) getSlow(pid int) interface{} {
    size := atomic.LoadUintptr(&p.localSize)
    locals := p.local
    // 循环尝试从其他P偷取对象，popTail从链表尾获取
    for i := 0; i < int(size); i++ {
        l := indexLocal(locals, (pid+i+1)%int(size))
        if x, _ := l.shared.popTail(); x != nil {
            return x
        }
    }

    size = atomic.LoadUintptr(&p.victimSize)
    if uintptr(pid) >= size {
        return nil
    }
    locals = p.victim
    l := indexLocal(locals, pid)
    if x := l.private; x != nil {
        l.private = nil
        return x
    }
    for i := 0; i < int(size); i++ {
        l := indexLocal(locals, (pid+i)%int(size))
        if x, _ := l.shared.popTail(); x != nil {
            return x
        }
    }

    // Mark the victim cache as empty for future gets don't bother
    // with it.
    atomic.StoreUintptr(&p.victimSize, 0)

    return nil
}

pushHead先取slot再判断是否可插入最后修改

func (d *poolDequeue) pushHead(val interface{}) bool {
    ptrs := atomic.LoadUint64(&d.headTail)
    head, tail := d.unpack(ptrs)
    if (tail+uint32(len(d.vals)))&(1<<dequeueBits-1) == head {
        // Queue is full.
        return false
    }
    slot := &d.vals[head&uint32(len(d.vals)-1)]

    // Check if the head slot has been released by popTail.
    typ := atomic.LoadPointer(&slot.typ)
    if typ != nil {
        // Another goroutine is still cleaning up the tail, so
        // the queue is actually still full.
        return false
    }

    // The head slot is free, so we own it.
    if val == nil {
        val = dequeueNil(nil)
    }
    *(*interface{})(unsafe.Pointer(slot)) = val

    // Increment head. This passes ownership of slot to popTail
    // and acts as a store barrier for writing the slot.
    atomic.AddUint64(&d.headTail, 1<<dequeueBits)
    return true
}

popTail先修改headTail再取slot然后重置slot

func (d *poolDequeue) popTail() (interface{}, bool) {
    var slot *eface
    for {
        ptrs := atomic.LoadUint64(&d.headTail)
        head, tail := d.unpack(ptrs)
        if tail == head {
            // Queue is empty.
            return nil, false
        }

        // Confirm head and tail (for our speculative check
        // above) and increment tail. If this succeeds, then
        // we own the slot at tail.
        ptrs2 := d.pack(head, tail+1)
        if atomic.CompareAndSwapUint64(&d.headTail, ptrs, ptrs2) {
            // Success.
            slot = &d.vals[tail&uint32(len(d.vals)-1)]
            break
        }
    }

    // We now own slot.
    val := *(*interface{})(unsafe.Pointer(slot))
    if val == dequeueNil(nil) {
        val = nil
    }

    // Tell pushHead that we're done with this slot. Zeroing the
    // slot is also important so we don't leave behind references
    // that could keep this object live longer than necessary.
    //
    // We write to val first and then publish that we're done with
    // this slot by atomically writing to typ.
    slot.val = nil
    atomic.StorePointer(&slot.typ, nil)
    // At this point pushHead owns the slot.

    return val, true
}

考虑正常push/pop情况，head和tail指向slot不会有交集，仅需原子获取修改headTail

考虑当发生写溢出，pop速度跟不上push速度时

  • 若pushHead先执行，在第一次计算队列时发现队列满，false退出
  • 若popTail先执行，tail成功+1
    • 此时pushHead执行，在第一次计算队列时发现队列没满，接着原子判断slot.typ
      • 若在判断前，popTail先执行原子设置slot.typ为空，则pushHead能获得slot正常插入，pop+push后队列仍满
      • 若pushHead先执行原子判断slot.typ，此时typ非空，push失败

func poolCleanup() {
    // Drop victim caches from all pools.
    for _, p := range oldPools {
        p.victim = nil
        p.victimSize = 0
    }

    // Move primary cache to victim cache.
    for _, p := range allPools {
        p.victim = p.local
        p.victimSize = p.localSize
        p.local = nil
        p.localSize = 0
    }

    // The pools with non-empty primary caches now have non-empty
    // victim caches and no pools have primary caches.
    oldPools, allPools = allPools, nil
}

  1. 初始状态，oldPools和allPools均为nil
  2. 第1次调用Get，由于p.local为nil，将会在pinSlow中创建p.local，然后将p放入allPools，此时allPools长度为1，oldPools为nil
  3. 对象使用完毕，第1次调用Put放回对象
  4. 第1次GC STW阶段，allPools中所有p.local将值赋值给victim并置为nil，最后allPools为nil，oldPools长度为1
  5. 第2次调用Get，由于p.local为nil，此时会从p.victim里面尝试取对象
  6. 对象使用完毕，第2次调用Put放回对象，但由于p.local为nil，重新创建p.local，并将对象放回，此时allPools长度为1，oldPools长度为1
  7. 第2次GC STW阶段，oldPools中所有p.victim置nil，前一次的cache在本次GC时被回收，allPools所有p.local将值赋值给victim并置为nil，最后allPools为 nil，oldPools长度为1

Go 1.13以前每次GC遍历allPools清空所有字段，新版相比Go 1.13以前，通过引入victim拉大了GC的粒度，且减小了GC的开销。
victim在GC时将当前对象放入其中，在下一次GC前仍可作为二级缓存获取数据，直到下一次GC时回收，即GC的粒度从原来每次GC回收空对象变为2次GC才回收空对象。同时由于从victim中获取的数据没有再次放回victim中，相应GC的开销也减小了。
Go 1.13通过增加缓存方式，每次对allPools对象一次性清空，相比以前使用循环遍历，GC时间复杂度由原来的O(n)变为O(1)

总结

  • 一个goroutine是一个g对象，main也是一个g
  • p数量默认为cpu核线程数，通过GOMAXPROCS调整
  • m数量一般比p多一点，最大10000个
  • goroutine入口newproc，m入口mstart
  • getg()获取g，g.m关联的m，g.m.p关联的p
  • 一般getg顺序为本地pop/全局拿最多32个/其他p偷一半(timer)/network
  • m通过schedule/findrunnable不断找活干，处于spinning状态。如果已经没活，退出spinning状态，调用stopm进入全局midle，同时释放p进入全局pidle
  • newproc中的g不会直接分配给m，而是由spinning的m自己找g处理。当有idle的p和非spinning的m，则将全局midle设置为spinning，然后找g处理。若m不足使用newm新创建m(比如程序刚开始启动时)
  • sysmon在main方法中注册，每个10ms检查g(初始20us逐渐增加到10ms，发生抢占后重置)，设置g.stackguard0=stackPreempt，在g运行过程中分配新stack时进行判断，将g放回全局runnable g队列，并调用一次schedule。每2分钟如果没有gc过，执行gc。若开启了strace，在sysmon中输出debug信息
  • gc根据上一次gc动态调整

func procresize(nprocs int32) *p {
    old := gomaxprocs

    // 1. Grow allp if necessary.
    if nprocs > int32(len(allp)) {
        // Synchronize with retake, which could be running
        // concurrently since it doesn't run on a P.
        lock(&allpLock)
        if nprocs <= int32(cap(allp)) {
            allp = allp[:nprocs]
        } else {
            nallp := make([]*p, nprocs)
            // Copy everything up to allp's cap so we
            // never lose old allocated Ps.
            copy(nallp, allp[:cap(allp)])
            allp = nallp
        }
        unlock(&allpLock)
    }

    // 2. initialize new P's
    for i := old; i < nprocs; i++ {
        pp := allp[i]
        if pp == nil {
            pp = new(p)
        }
        pp.init(i)
        atomicstorep(unsafe.Pointer(&allp[i]), unsafe.Pointer(pp))
    }

    _g_ := getg()

    // 2. 增加procs，复用g的p
    if _g_.m.p != 0 && _g_.m.p.ptr().id < nprocs {
        // continue to use the current P
        _g_.m.p.ptr().status = _Prunning
        _g_.m.p.ptr().mcache.prepareForSweep()

    // 3. 减少procs且当前g的p比新值小，释放当前g的p，使用allp[0]
    } else {
        if _g_.m.p != 0 {
            _g_.m.p.ptr().m = 0
        }
        _g_.m.p = 0
        _g_.m.mcache = nil
        p := allp[0]
        p.m = 0
        p.status = _Pidle
        acquirep(p)
    }

    // 4. 调小procs，释放nprocs～old的p资源
    for i := nprocs; i < old; i++ {
        p := allp[i]
        p.destroy()
        // can't free P itself because it can be referenced by an M in syscall
    }

    // Trim allp.
    if int32(len(allp)) != nprocs {
        lock(&allpLock)
        allp = allp[:nprocs]
        unlock(&allpLock)
    }

    // 5. 关联有g的p和空闲m，空g的p放入全局空p对象，并返回第一个可用p
    var runnablePs *p
    for i := nprocs - 1; i >= 0; i-- {
        p := allp[i]
        if _g_.m.p.ptr() == p {
            continue
        }
        p.status = _Pidle
        if runqempty(p) {
            pidleput(p)
        } else {
            p.m.set(mget())
            p.link.set(runnablePs)
            runnablePs = p
        }
    }
    stealOrder.reset(uint32(nprocs))
    var int32p *int32 = &gomaxprocs // make compiler check that gomaxprocs is an int32
    atomic.Store((*uint32)(unsafe.Pointer(int32p)), uint32(nprocs))
    return runnablePs
}

newproc

newproc是goroutine入口(main也是goroutine)

  1. 获取free g
    1.1 本地free g队列获取g
    1.2 若本地free g队列空，全局free g队列获取最多32个g到本地，优先取有栈g
    1.3 若全局free g队列空，创建新g，默认stack为2k，状态默认idle->dead，避免gc
  2. 栈信息拷贝
  3. 状态dead->runnable
  4. 从goidcache设置goid，采用了预分配id段，预分配加解锁一次，设置无需锁
  5. 保存g到p
    5.1 保存g到runnext，旧的runnext排到本地runnable g队列尾(大小固定256)
    5.2 若本地runnable g队列满，取一半放到全局runnable g队列
  6. 有空闲p，并且没有spinning状态m，调用wakep消化g。 main函数也是一个goroutine，通过mainStarted过滤，无需执行wakep
    6.1 wakep调用startm，首先查看有无全局midle，若有直接绑定gpm，若没有newm创建一个spinning的m，由m调用mstart找g处理
func newproc(siz int32, fn *funcval) {
    argp := add(unsafe.Pointer(&fn), sys.PtrSize)
    gp := getg() // debug trace用
    pc := getcallerpc()
    systemstack(func() {
        newproc1(fn, argp, siz, gp, pc)
    })
}

func newproc1(fn *funcval, argp unsafe.Pointer, narg int32, callergp *g, callerpc uintptr) {
    _g_ := getg()

    acquirem()

    _p_ := _g_.m.p.ptr()

    newg := gfget(_p_) // 1. 获取free g

    // 如果本地g和全局g都用完，创建新g，newg=new(g)
    // _StackMin=2048，因此默认创建出来的g栈大小为2k
    if newg == nil {
        newg = malg(_StackMin)
        casgstatus(newg, _Gidle, _Gdead) // 状态_Gidle -> _Gdead
        allgadd(newg) // publishes with a g->status of Gdead so GC scanner doesn't look at uninitialized stack.
    }

    // 2. 拷贝栈信息

    newg.ancestors = saveAncestors(callergp) // 用于go trace

    casgstatus(newg, _Gdead, _Grunnable) // 3. 状态_Gdead -> _Grunnable

    // 4. 防止每次访问全局计数器goidgen，采用预分配id段方式
    if _p_.goidcache == _p_.goidcacheend {
        _p_.goidcache = atomic.Xadd64(&sched.goidgen, _GoidCacheBatch)
        _p_.goidcache -= _GoidCacheBatch - 1
        _p_.goidcacheend = _p_.goidcache + _GoidCacheBatch
    }
    newg.goid = int64(_p_.goidcache)
    _p_.goidcache++

    // 5. 保存g到p
    runqput(_p_, newg, true)

    // 6. 有空闲p，并且没有spinning状态m，重新唤醒p。 main函数也是一个goroutine，通过mainStarted过滤，无需执行wakep
    if atomic.Load(&sched.npidle) != 0 && atomic.Load(&sched.nmspinning) == 0 && mainStarted {
        wakep()
    }

    releasem(_g_.m)
}

schedule是单次调度的公共函数，找一个g放m上执行，调用excute

  1. 有locked g，直接在上面跑execute
  2. 61轮一次从全局runnable g取 (增加循环间隔，避免全局g被某几个goroutine占有)
  3. 从本地runnable g取，优先取runnext
  4. 调用findrunnable()获取runnable g，直到取到1个g为止
  5. 若当前m处于spinning，取消spinning
  6. 获取的g已有lockedm，绑定g和lockedm，从2.重新判断
func schedule() {
    ...

    // 1. 有locked g，直接在上面跑execute
    if _g_.m.lockedg != 0 {
        stoplockedm()
        execute(_g_.m.lockedg.ptr(), false) // Never returns.
    }

top:
    pp := _g_.m.p.ptr()
    pp.preempt = false

    if sched.gcwaiting != 0 {
        gcstopm()
        goto top
    }

    var gp *g
    var inheritTime bool

    // 2. 61轮一次从全局runnable g取
    if gp == nil {
        // Check the global runnable queue once in a while to ensure fairness.
        // Otherwise two goroutines can completely occupy the local runqueue
        // by constantly respawning each other.
        if _g_.m.p.ptr().schedtick%61 == 0 && sched.runqsize > 0 {
            lock(&sched.lock)
            gp = globrunqget(_g_.m.p.ptr(), 1)
            unlock(&sched.lock)
        }
    }

    // 3. 从本地runnable g取，优先取runnext
    if gp == nil {
        gp, inheritTime = runqget(_g_.m.p.ptr())
    }
    // 4. 调用findrunnable()获取runnable g，直到取到1个g为止
    if gp == nil {
        gp, inheritTime = findrunnable() // blocks until work is available
    }

    // 5. 若当前m处于spinning，取消spinning
    if _g_.m.spinning {
        resetspinning()
    }

    // 6. 获取的g已有lockedm，绑定g和lockedm，从2.重新判断
    if gp.lockedm != 0 {
        // Hands off own p to the locked m,
        // then blocks waiting for a new p.
        startlockedm(gp)
        goto top
    }

    execute(gp, inheritTime)
}

findrunnable()尝试从本地g/全局g/net poll/其他p偷取g或timer g/执行gc 各种途径find a work

  1. 再尝试从本地runnable g取
  2. 从全局runnable g取
  3. 从net poll取g
  4. 大量m已处于spinning，而非idle的p数量少，通常为代码并发度低，跳过5.
  5. 从其他p偷取一半本地runnable g，尝试4次，最后一次会尝试偷取runNext和p2的到期timer g
    5.1 若有timer，可能有timer g结束入队，重新从1.开始判断
  6. gc阶段，查看有无gc工作可做
  7. 再尝试从全局runnable g取
  8. 释放p，放回全局pidle
    8.1 m状态变为非spinning，nmspinning计数-1
  9. 从allp重新获取idle的p，并设置spinning和nmspinning+1，重新从1.开始判断
  10. 处于gc阶段，重新从6.开始判断
  11. 再次从net poll取g
  12. 将m放入全局idle m，并与当前g关联，重新从1.开始判断
func findrunnable() (gp *g, inheritTime bool) {
    _g_ := getg()

top:
    _p_ := _g_.m.p.ptr()

    // 1. 再尝试从本地runnable g取
    if gp, inheritTime := runqget(_p_); gp != nil {
        return gp, inheritTime
    }

    // 2. 从全局runnable g取
    if sched.runqsize != 0 {
        lock(&sched.lock)
        gp := globrunqget(_p_, 0)
        unlock(&sched.lock)
        if gp != nil {
            return gp, false
        }
    }

    // 3. 从net poll取g

    procs := uint32(gomaxprocs)
    ranTimer := false
    // If number of spinning M's >= number of busy P's, block.
    // 4. 大量m已处于spinning，而非idle的p数量少，通常为代码并发度低
    if !_g_.m.spinning && 2*atomic.Load(&sched.nmspinning) >= procs-atomic.Load(&sched.npidle) {
        goto stop
    }
    if !_g_.m.spinning {
        _g_.m.spinning = true // 设置为spinning
        atomic.Xadd(&sched.nmspinning, 1)
    }

    // 5. 从其他p偷取一半本地runnable g，尝试4次，最后一次会尝试偷取runNext和p2的到期timer g
    for i := 0; i < 4; i++ {
        for enum := stealOrder.start(fastrand()); !enum.done(); enum.next() {
            if sched.gcwaiting != 0 {
                goto top
            }
            stealRunNextG := i > 2 // first look for ready queues with more than 1 g
            p2 := allp[enum.position()]
            if _p_ == p2 {
                continue
            }
            if gp := runqsteal(_p_, p2, stealRunNextG); gp != nil {
                return gp, false
            }

            // 偷取p2的timer g
            if i > 2 && shouldStealTimers(p2) {
                tnow, w, ran := checkTimers(p2, now)
                now = tnow
                if w != 0 && (pollUntil == 0 || w < pollUntil) {
                    pollUntil = w
                }
                if ran {
                    // p2的timer g入队，可以偷取
                    if gp, inheritTime := runqget(_p_); gp != nil {
                        return gp, inheritTime
                    }
                    ranTimer = true
                }
            }
        }
    }

    // 5. 若有timer，可能有timer g结束入队，重新从1.开始判断
    if ranTimer {
        // Running a timer may have made some goroutine ready.
        goto top
    }

stop:
    // 6. gc阶段，查看有无gc工作可做
    if gcBlackenEnabled != 0 && _p_.gcBgMarkWorker != 0 && gcMarkWorkAvailable(_p_) {
        _p_.gcMarkWorkerMode = gcMarkWorkerIdleMode
        gp := _p_.gcBgMarkWorker.ptr()
        casgstatus(gp, _Gwaiting, _Grunnable)
        if trace.enabled {
            traceGoUnpark(gp, 0)
        }
        return gp, false
    }

    // 7. 再尝试从全局runnable g取
    if sched.runqsize != 0 {
        gp := globrunqget(_p_, 0)
        unlock(&sched.lock)
        return gp, false
    }

    // 8. 释放m的p，放回全局pidle
    if releasep() != _p_ {
        throw("findrunnable: wrong p")
    }
    pidleput(_p_)
    unlock(&sched.lock)

    // 8.1 m状态变为非spinning，nmspinning-1
    wasSpinning := _g_.m.spinning
    if _g_.m.spinning {
        _g_.m.spinning = false
        if int32(atomic.Xadd(&sched.nmspinning, -1)) < 0 {
            throw("findrunnable: negative nmspinning")
        }
    }

    // 9. 从allp重新获取idle的p，并设置spinning和nmspinning+1，重新从1.开始判断
    for _, _p_ := range allpSnapshot {
        if !runqempty(_p_) {
            lock(&sched.lock)
            _p_ = pidleget()
            unlock(&sched.lock)
            if _p_ != nil {
                acquirep(_p_)
                if wasSpinning {
                    _g_.m.spinning = true
                    atomic.Xadd(&sched.nmspinning, 1)
                }
                goto top
            }
            break
        }
    }

    // 10. 处于gc阶段，重新从6.开始判断

    // 11. 再次从net poll取g

    // 12. 将m放入全局idle m，并与当前g关联，重新从1.开始判断
    stopm()
    goto top
}

sysmon为死循环，调用retake来调度长时间执行的g；调用gcTrigger.test方法判断2分钟没执行过gc；如果设置strace，根据设置间隔输出debug信息

func sysmon() {
    lock(&sched.lock)
    sched.nmsys++
    checkdead()
    unlock(&sched.lock)

    lasttrace := int64(0)
    idle := 0
    delay := uint32(0)
    for {
        if idle == 0 { // start with 20us sleep...
            delay = 20
        } else if idle > 50 { // start doubling the sleep after 1ms...
            delay *= 2
        }
        if delay > 10*1000 { // up to 10ms
            delay = 10 * 1000
        }
        usleep(delay)
        now := nanotime()
        // retake P's blocked in syscalls
        // and preempt long running G's
        if retake(now) != 0 {
            idle = 0
        } else {
            idle++
        }
        // check if we need to force a GC，判断在t.test()中
        if t := (gcTrigger{kind: gcTriggerTime, now: now}); t.test() && atomic.Load(&forcegc.idle) != 0 {
            lock(&forcegc.lock)
            forcegc.idle = 0
            var list gList
            list.push(forcegc.g)
            injectglist(&list)
            unlock(&forcegc.lock)
        }
        if debug.schedtrace > 0 && lasttrace+int64(debug.schedtrace)*1000000 <= now {
            lasttrace = now
            schedtrace(debug.scheddetail > 0)
        }
    }
}

retake遍历allp判断执行超过10ms的p，传给preemptone

const forcePreemptNS = 10 * 1000 * 1000 // 10ms

func retake(now int64) uint32 {
    n := 0
    lock(&allpLock)
    for i := 0; i < len(allp); i++ {
        _p_ := allp[i]
        if _p_ == nil {
            // 在procresize代码中，还没初始化
            continue
        }
        pd := &_p_.sysmontick
        s := _p_.status
        sysretake := false
        if s == _Prunning || s == _Psyscall {
            // Preempt G if it's running for too long.
            t := int64(_p_.schedtick)
            if int64(pd.schedtick) != t {
                pd.schedtick = uint32(t)
                pd.schedwhen = now
            } else if pd.schedwhen+forcePreemptNS <= now {
                preemptone(_p_)
                sysretake = true
            }
        }
    }
    unlock(&allpLock)
    return uint32(n)
}

func preemptone(_p_ *p) bool {
    mp := _p_.m.ptr()
    gp := mp.curg
    gp.preempt = true

    // Every call in a go routine checks for stack overflow by
    // comparing the current stack pointer to gp->stackguard0.
    // Setting gp->stackguard0 to StackPreempt folds
    // preemption into the normal stack overflow check.
    gp.stackguard0 = stackPreempt

    return true
}
